uchroma.server.input module¶

class uchroma.server.input.InputManager(driver, input_devices: list)[source]¶

Bases: object

Manages event devices associated with a physical device instance and allows for callback registration. Reader loop is fully asynchronous. See the InputQueue class for a higher level API.

add_callback(callback) → bool[source]¶

Add a new callback (coroutine) which will fire when new input events are received.

Return type:bool
Parameters:callback – coroutine to add
Returns:True if successful
grab(excl: bool)[source]¶

Get exclusive access to the device

WARNING: Calling this on your primary input device might cause you a bad day (or at least a reboot)! Use with devices (like keypads) where we don’t want other apps to see actual scancodes.

Parameters:excl (bool) – True to gain exclusive access, False to release
input_devices¶

List of input devices associated with the parent device
